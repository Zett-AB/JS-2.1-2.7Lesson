2.1 Подключение JS к странице и переменные

Подключение JS к странице и переменные

JavaScpirt(JS) - является высокоуровневым языком, браузер его читает таким какой он есть.

Существуют :

  • локальные файлы - хранятся/лежат на компьютере;

  • удаленные файлы - хранятся на сервере.

Как правило простой скрипт/код прописывают прямо в HTML, например:
body>
script> команда/код скрипта /script>
/body>
Код со скриптом рекомендуется располагать в футере страницы.

Сложный скрипт - код пишут и сохраняют в отдельном файле и затем подключают к странице.
Пример подключения JS файла к странице HTML:
body>
script src="script.js"> /script>
/body>

При написании кода на JS:
// - однострочный комментарий /*.....*/- много строчный комментарий.

Переменная - это храние какой-либо информации.
Существует три способа объявления переменной в JS:

  • var

  • let

  • const

Пример классического объявления переменной:
var leftBorderWidth=1;
где
var - ключевое слово указывающее, что будет переменная и var уже виден в коде.
leftBorderWidth - имя переменной

  • имя переменной пишется только латиницей, нижнее подчеркивание, цифры и знак $;

  • имя переменной всегда должно быть осмыленным, т.е. имя переменной указывает какие именно данные хранятся в переменной;

  • имя переменной не может начинаться с цифры.

= - это присвоение значения переменной.
1 - присвоенной значение переменной или проще говоря 1 - это значение переменной.

let second = 2;
где
let - ключевое слово, её отличие от var в том, что данная переменная создается тогда, когда код до неё доходит, что называется ещё хостинг или всплытие.
Все остальное также как в var.

const pi=3,14;
где
const - ключевое слово, эта переменная создает константу, т.е. неизменное значение переменной. Проще говоря, когда мы создаем переменную с помощью const, то её значение неизменно.

ВАЖНО ЗНАТЬ

Важно знать, что в настоящее время ключевое слово var при объявлении переменной не используется.

2.2 Классификация типов данных

Классификация типов данных

В JS два типа данных:

  • Простые их 7;

  • Объект

Простые(примитивные)

Простые(примитивные) типы данных:

  1. Число
    Числовой тип данных(number) представляет собой как целочисленное значение(например let n = 9;), так и числа с плавающей точкой(например let n = 3.14;).
    Кроме обычных чисел существуют также "специальные числовые значения"

    • Infinity представляет собой бесконечность. Это особое значение, которое больше любого числа. Его можно получить при делении на ноль.(например alert(1/0);)

    • NaN означает вычислительную ошибку. Это результат нерпавильной или неопределенной математической операции.(например alert("не число" /2);)

  2. BigInt
    В JS тип number не может безопасно работать с числами, большими, чем (253-1) (т. е. 9007199254740991) или меньшими, чем -(253-1) для отрицательных чисел.
    Когда нужен диапазон действительно гигантских целых чисел без каких-либо ограничений или пропущенных значений внутри него. Например, в криптографии или при использовании метки времени («timestamp») с микросекундами.
    Тип BigInt был добавлен в JavaScript, чтобы дать возможность работать с целыми числами произвольной длины.
    Чтобы создать значение типа BigInt, необходимо добавить n в конец числового литерала:
    // символ "n" в конце означает, что это BigInt
    const bigInt = 1234567890123456789012345678901234567890n;

  3. Строка
    Строка (string) в JS должна быть заключена в кавычки.
    let str="Привет";
    let str2 = 'Одинарные кавычки тоже подойдут';
    let phrase = `Обратные кавычки позваляют вставлять переменные $(str)`;
    То есть мы видим, что в JS есть три типа кавычек:

    • двойные "";

    • одинарные '';

    • обратные ``;

    Двойные и одинарные кавычки являются простыми и между ними нет разницы в JS.
    Обратные же кавычки имеют расширенную функциональность. Они позволяют нам встраивать выражения в строку, заключая их в ${…}. Например:
    let name = "Иван";
    // Вставим переменную
    alert( `Привет, ${name}!` ); // Привет, Иван!
    // Вставим выражение
    alert( `результат: ${1 + 2}` ); // результат: 3

  4. Логический(Булевый)
    Булевый тип (boolean) может принимать только два значения: true (истина) и false (ложь).
    Такой тип, как правило, используется для хранения значений да/нет: true значит «да, правильно», а false значит «нет, не правильно».
    Например:
    let nameFieldChecked = true; // да, поле отмечено
    let ageFieldChecked = false; // нет, поле не отмечено

  5. Филосовский(значение null)
    Специальное значение null не относится ни к одному из типов, описанных выше.
    Оно формирует отдельный тип, который содержит только значение null:
    let age = null;
    В JavaScript null не является «ссылкой на несуществующий объект» или «нулевым указателем», как в некоторых других языках.
    Это просто специальное значение, которое представляет собой «ничего», «пусто» или «значение неизвестно».
    В приведённом выше коде указано, что значение переменной age неизвестно.

  6. undefined
    Специальное значение undefined также стоит особняком. Оно формирует тип из самого себя так же, как и null.
    Оно означает, что «значение не было присвоено».
    Если переменная объявлена, но ей не присвоено никакого значения, то её значением будет undefined:
    let age;
    alert(age); // выведет "undefined"
    Технически мы можем присвоить значение undefined любой переменной:
    let age = 123;
    // изменяем значение на undefined
    age = undefined;
    alert(age); // "undefined"
    …Но так делать не рекомендуется.
    Обычно null используется для присвоения переменной «пустого» или «неизвестного» значения, а undefined – для проверок, была ли переменная назначена.

  7. Символ(symbol)
    Используется редко.
    Тип symbol (символ) используется для создания уникальных идентификаторов в объектах.
    let sym = Symbol();

  8. Объект(object)
    Объекты занимают важное место в языке и требуют особого внимания.
    let obj = {};
    это коллекция данных, эта структура, которая используется для хранения любых данных.
    Пример:
    let person = {
    name: "Джон",
    age: "25",
    isMerried: false
    };
    массив закрывается в квадратные скобки [...] и каждому значению присваивается порядковый номер.
    Пример:
    let arr = ['plum.png','orange.jpg','apple.bmp'];
    в данном примере нумеруется весь массив данных начиная с plum.png, которому присваивается номер 0 и заканчивая apple.bmp, которму присваивается номер 2(так как номерация всегда начинается с 0).

2.3 Простое общение с пользователем

Взаимодействие с пользователем

Для дальнейшего изучения нам необходимо создать файл script.js, в который мы будем писать наши первые простые команды и проверять их в консоли нашего редактора VSCode.
Здесь мы будем дублировать наш код и писать к нему пояснения.

ВАЖНО!
После того как мы создали новый документ script.js в самой первой строке кода мы обязательно должны написать директиву 'use strict';.
Это важная директива она указывает, что мы пишем код на стандарте ES6 JS.
Директива выглядит как строка: "use strict"; или 'use strict';. Когда она находится в начале скрипта, весь сценарий работает в «современном» режиме.
Данную директиву необходимо писать или в начале скрипта или на первой строке когда пишем код в отдельном файле.

Когда мы пишем код мы должны знать, что основная команда разработчика кода это:
console.log();
Данная команда выводить информацию соответственно в консоль и таким образом, мы проверяем правильно ли работает код.
Давайте для проверки напишем следующий код в нашем уже созданом файле и проверим его с помощью данной команды.
И так начинаем:
console.log(4/0); // получим Infinity
Теперь создадим переменную persone и создадим для неё несколько данных, которые потом выведем через вышеуказанную команду console.log();
let persone ={
name: "John",
age: 25,
isMarried: false
};
console.log(persone["name"]);
На выходе видим John.
Теперь прдыдущий код комментим и пишем новый:
let arr = ['plum.png', 'orange.jpg','apple.bmp'];
console.log(arr[0]);
На выходе видим, что индекс 0 в указанном массиве имеет значение plum.png.

В массиве каждому значению присваивается свой индекс.
Присвоение индекса начинается всегда с 0.

К сожалению не все команды можно проверить в редакторе кода, но весь код всегда можно проверить в консоли браузера.
Поэтому некоторый код мы будем писать в консоли браузера и смотреть результат.
Кроме того, написанный код мы можем проверить подключив наш файл со скриптом к основному файлу. В нашем случаи мы подключаем наш файл script.js к нашему основному файлу index.html и теперь когда будем открывать наш основной файл в браузере, то наш скрипт в файле script.js будет срабатывать.
Напоминаю, что подключать наш файл script.js, лучше всего после футера и для подключения файла пишем следующий код script src="file_name">/script>. У нас имя файла(file_name) script.js

Переходим непосредственно к командам взаимодействия с пользователем.
Открываем окно браузера и запускаем консоль разработчика нажав клавишу F12.
Выбираем вкладку "консоль" и уже в ней пишем код и смотрим как он работает.
Или подключаем наш файл script.js к нашему файлу index.html и тогда смотрим работу скрипта.

Рассмотрим базовые функции, которые будем использовать в браузере, а именно:

  • alert;

  • prompt;

  • confirm;

Начнем с функции alert.
Она выводит на экран окно с сообщением и приостанавливает выполнение скрипта, пока пользователь не нажмёт кнопку «ОК».
При выполнении срипта происходит выводт небольшого окна с сообщением, которе называется "модальным окном".
Понятие модальное означает, что пользователь не может взаимодействовать с интерфейсом остальной части страницы, нажимать на другие кнопки и т.д. до тех пор, пока взаимодействует с окном.
В данном случае – пока не будет нажата кнопка «OK».

Синтаксис :
alert("сообщение");

Напишем следующий код:
alert("Hello World");
alert - это команда;
в скобках в кавычках указываем то, что хотим вывести на экран браузера, в данном случаи будет табличка с приветствием "Hello World".
При этом, помним, что все остальные действия скрипта приостанавливаются до закрытия окошка.

Для наглядного примера добавми еще одну строку в наш код:
alert("Привет");
Обновляем браузер и видим, что пока не закрываем окно с "Hello World", не появляется второе окно с "Привет".

Функция prompt
Функция prompt принимает два аргумента.
Синтаксис:
result = prompt(title, [default]);
Этот код отобразит модальное окно с текстом, полем для ввода текста и кнопками OK/Отмена.
где
title - текст для отображения в окне,
default - необязательный второй параметр, который устанавливет начальное значение в поле для текста в окне.

Квадратные скобки в синтаксисе [...]
Квадратные скобки вокруг default в описанном выше синтаксисе означают, что параметр факультативный, необязательный.

Пользователь может напечатать что-либо в поле ввода и нажать OK.
Введённый текст будет присвоен переменной result.
Пользователь также может отменить ввод нажатием на кнопку «Отмена» или нажав на клавишу Esc.
В этом случае значением result станет null.
Вызов prompt возвращает текст, указанный в поле для ввода, или null, если ввод отменён пользователем.
Результат действий пользователя мы можем увидить в консоли браузера.
Ответ пользователя всегда будет строка(string), даже если он наберет цифры. Чтобы это проверить вводим conlose.log(typeof(переменная));

Несколько примеров:
let age = prompt('Сколько Вам лет?', 100);
alert('Тебе '+ age + ' лет!');

Что мы написали в коде, в первой строке кода мы создали переменную age в которой в качестве значения прописали функцию promt, которая выведет модальное окно с вопросом - Сколько Вам лет?.
Вторым дополнительным условием стоит цифра 100, которая отобразится в поле ввода.
Во второй строке мы написали функцию alert с переменной age и которая должна нам вывести, после того как мы ответим на первое модальное окно, модальное окно содержащим - Тебе (переменная age(значение которое мы введем в поле)) лет!.

Теперь пишем этот код в нашем файле и проверяем.

Если мы не станем заполнять поле и нажмем кнопку Отменить, то в ответе увидим - Тебе null лет. Таким образом, из ответа следует,что ответа нет(несуществует), т.е. пользователь не ответил на вопрос.
Если теперь мы в консоли браузера напишем console.log(typeof(age)), то увидим ответ string.
Какие бы значения мы не вводили в строку ответ всегда будет string.

Теперь давайте напишем другой код и посмотрим какой ответ будет в консоли.
Пишем следующий код:
let answer = prompt("Есть ли Вам 18 лет?", "Да");
В модальном окне мы увидим - Есть ли Вам 18 лет?
В строке будет - Да
Теперь мы снова в консоли браузера напишем console.log(typeof(answer)), то увидим ответ string.
С помощью функции typeof() мы проверяем какой тип значения у переменной.

Функция confirm
Функция confirm отображает модальное окно с текстом вопроса question и двумя кнопками: OK и Отмена.
Синтаксис:
result = confirm(question);
где
переменная result содержит значение в виде функции comfirm с вопросом.
Результат – true, если нажата кнопка OK. В других случаях – false.
Результат мы увидим в консоли.

Давайте проверим и пишем код:
let secAge= confirm(Are you here?);
alert(secAge);
В браузере появиться модальное окно с вопросом - Are you here?
Если вы ответите - да, появится следующее модальное окно с надписью true, если нажать отмена в окошке будет false.
Если мы уберем из кода вторую строку - alert(secAge);, то проверить какой был ответ пользователя мы можем через консоль командой console.log(secAge); и увидим результат либо true, либо false.

Мы рассмотрели 3 функции браузера для взаимодействия с пользователем:

alert
показывает сообщение.

prompt
показывает сообщение и запрашивает ввод текста от пользователя.
Возвращает напечатанный в поле ввода текст или null, если была нажата кнопка «Отмена» или Esc с клавиатуры.

confirm
показывает сообщение и ждёт, пока пользователь нажмёт OK или Отмена.
Возвращает true, если нажата OK, и false, если нажата кнопка «Отмена» или Esc с клавиатуры.

Все эти методы являются модальными: останавливают выполнение скриптов и не позволяют пользователю взаимодействовать с остальной частью страницы до тех пор, пока окно не будет закрыто.

На все указанные методы распространяются два ограничения:
Расположение окон определяется браузером. Обычно окна находятся в центре.
Визуальное отображение окон зависит от браузера, и мы не можем изменить их вид.
Такова цена простоты.
Есть другие способы показать более приятные глазу окна с богатой функциональностью для взаимодействия с пользователем, но если «навороты» не имеют значения, то данные методы работают отлично.

2.4 Операторы JavaScpirt

Операторы

Основные определения

Выражения в JavaScpirt
представляют собой комбинации операндов и операторов.

Операции в выражениях выполняются последовательно в соответствии со значением приоритета(чем больше значение приоритета, тем он выше).
Возвращаемый результат не всегда имеет значение того же типа, что и тип обрабатываемых данных.
Как пример - в операциях сравнения участвуют операнды различных типов, но возвращаемый результат всегда будет логического типа.

Запишем код:
var x = 50 + "day";
где:
x - переменная;
= и + - операторы;
50 и "day" - операнды;

Операнды - это данные, обрабатываемые сценарием JavaScript.
В качестве операндов могут быть как простые типы данных, так и сложные, а также другие выражения.

Операторы - это символы языка, выполняющие различные операции с данными.
Операторы могут записываться с помощью символов пунктуации или ключевых слов.

В зависимости от количества операндов различают следующие типы операторов:
униарный - в операции участвует один операнд;
бинарный - в операции участвуют два операнда;
тернарный - комбинирует три операнда.

Простейшая форма выражения — литерал — нечто, вычисляемое само в себя, например, число 100, строка "Hellow world".
Переменная тоже может быть выражением, так как она вычисляется в присвоенное ей значение.

Краткий перечень видов операторов

  • Арефметические операторы;

  • Операторы присваивания;

  • Операторы инкремента и дискримента;

  • Операторы сравнения;

  • Логические операторы;

  • Побитовые операторы;

  • Строковые операторы;

  • Специальные операторы;

  • Комментарии в JavaScpirt.

Арифметические операторы

Данные операторы предназначены для выполнения математических операций, они работают с числовыми операндами (или переменными, хранящими числовые значения), возвращая в качестве результата числовое значение.
Если один из операндов является строкой, интерпретатор JavaScript попытается преобразовать его в числовой тип, а после выполнить соответствующую операцию. Если преобразование типов окажется невозможным, будет получен результат NaN (не число).

К данным операторам относятся:

  • Сложение (+)- складывает числовые операнды.
    Если один из операндов — строка, то результатом выражения будет строка.
    Приоритет - 12;

  • Вычитание(-) - выполняет вычитание второго операнда из первого.
    Приоритет - 12;

  • Унарный минус(-)- преобразует положительное число в отрицательное, и наоборот.
    Приоритет - 14;

  • Умножение(*)- умножает два операнда.
    Приоритет - 13;

  • Деление(/)- делит первый операнд на второй.
    Результатом деления может являться как целое, так и число с плавающей точкой.
    Приоритет - 13;

  • Деление по модулю(%) - вычисляет остаток, получаемый при целочисленном делении первого операнда на второй.
    Применяется как к целым числам, так и числам с плавающей точкой.
    Приоритет - 13;

  • Возведение в степень(**) - оператор возведения a ** b
    т.е. возведет a в степнь b.

Примеры арифметических операторов

Пишем следующий код:

let x=5, y=8, z;
z=x+y;// вернет 13
alert(z);//выводим на экран браузера значение переменной х

let x=5, y=8, z;
z=x-y;// вернет -3
alert(z);//выводим на экран браузера значение переменной х

let x=5, y=8, z;
z= -y;// вернет -8
alert(z);//выводим на экран браузера значение переменной х

let x=5, y=8, z;
z=x*y;// вернет 40
alert(z);//выводим на экран браузера значение переменной х

let x=5, y=8, z;
z=x/y;// вернет 0.625
alert(z);//выводим на экран браузера значение переменной х

let x=5, y=8, z;
z=y%x;// вернет 3
alert(z);//выводим на экран браузера значение переменной х

Конкатенация

Конкатенация - это процесс соединения вещей друг с другом.
В JavaScript конкатенация чаще всего используется для соединения значений переменных друг с другом, или строк со строками (для образования более длинных строк)

Оператор плюс (+) обычно суммирует числа, но если его применить к строкам, то он их объединяет(склеивает).
Пример:
let s="моя" + " строка";
alert(s);
В браузере видим - моя строка.

Обратите внимание, что если какой-либо из операндов является строкой, то другой тоже преобразуется в строку.
Например:
alert('1'+2);//12
alert(2+'1');//21

alert(2 + 2 + '1' ); // "41" а не "221"
В данном примере операторы работают один за другим.
Первое +суммирует два числа, поэтому оно возвращает4, затем следующее +добавляет 1к нему строку, так что это похоже 4 + '1' = '41'.

alert('1' + 2 + 2); // "122" а не "14"
В данном примере первый операнд представляет собой строку, компилятор обрабатывает два других операнда также как строки. Они 2 объединяются с '1', так что это похоже '1' + 2 = "12", и "12" + 2 = "122".
Двоичный (+) - единственный оператор, который поддерживает строки таким образом.
Другие арифметические операторы работают только с числами и всегда преобразуют свои операнды в числа.

Вот демонстрация для вычитания и деления:
alert( 6 - '2' ); // 4, конвертируется строка '2' в число
alert( '6' / '2' ); // 3, конвертация всех операндов в числа

Унарное +

Оператор плюс(+) также существует в унарной форме.
Унарный плюс(+) или, другими словами, оператор плюс(+), применяемый к одному значению, ничего не делает с числами.
Но если операнд не является числом, унарный плюс(+) преобразует его в число.
Например:
// Не преобразует в число
let x = 1;
alert( +x ); // 1
let y = -2;
alert( +y ); // -2

// конвертирует в число
alert( +true ); // 1
alert( +"" ); // 0

Необходимость преобразования строк в числа возникает очень часто.
Например, если мы получаем значения из полей формы HTML, они обычно являются строками.
Что, если мы хотим их суммировать?
Двоичный плюс добавит их в виде строк:
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", пример объединения строк

Если мы хотим рассматривать их как числа, нам нужно преобразовать, а затем суммировать их:
let apples = "2";
let oranges = "3";
// оба значения преобразуются в числа перед двоичным знаком плюс alert( +apples + +oranges ); // 5 // более длиный вариант // alert( Number(apples) + Number(oranges) ); // 5

С точки зрения математика, обилие плюсов может показаться странным.
Но с точки зрения программиста, в этом нет ничего особенного: сначала применяются унарные плюсы, они преобразуют строки в числа, а затем двоичный плюс суммирует их.
Почему унарные плюсы применяются к значениям перед двоичными?
Как мы увидим, это связано с их более высоким приоритетом.

Операторы присваивания

Операторы присваивания используются для присваивания значений переменным.
Комбинированные операторы позволяют сохранить первоначальное и последующее значение в одной переменной.

Операторы присвоения:

  • Присвоение(=) - используется для присваивания значения переменной;
    Приоритет - 2;

  • Комбинированный оператор(+=, -=, *=, /=, %=,**=) - выполняет присваивание с операцией.
    Между первым и вторым операндом выполняется соответствующая операция, затем результат присваивается первому операнду.

Примеры применения операторов присвоения:
let a = 5; //переменной а присвоиваем числовое значение 5;
alert(a); // выводим на экран значение присвоенное переменной а;
let b = "Привет"; // присвоение перменнной b значения строчка - Привет
let m = n = z = 8; // присовоение переменным m, n и z числового значения 8. Присвоение идет справа на лево, т.е. сначала числовое значение присваивается переменной z, затем переменной n присваивается в качестве значения переменная z имеющая значение 8, и переменной m присваивается в качестве значения переменная n.

Примеры комбинированного присвоения:
x += 10; // равнозначно x = x + 10;
x -= 10; // равнозначно x = x - 10;
x *= 10; // равнозначно x = x * 10;
x /= 10; // равнозначно x = x / 10;
x %= 10; // равнозначно x = x % 10;
x **=10 // равнозначно x = x**10(т.е. возводим 10 в 10 степень)

На практике комбинованное присвоение выглядит следующим образом:
let x = 1;
x += 10;
alert(x);
на экране будет 11
или другой вариант
let d = 5;
let f = 2;
d +=f;
alert(d);
на экране видим 7

Можно выполнить комбинированное присвоение и другим образом:
let x = 10;
x += 10;
alert(x + ' - это х+=10, где х=10');
x -=10;
alert(x + ' - это х-=10, здесь уже х=20');
x *=10;
alert(x + ' - это х*=10, здесь уже х=10');
x /=10;
alert(x + ' - это х/=10, здесь уже х=100');
x %=10;
alert(x + ' - это х%=10, здесь х=10');
В данном примере мы работаем с переменной х, значение которой меняется взависимости от проведенной с переменной операции присвоения.
Таким образом, после первой операции значении переменной х = 20 и соответственно во второй операции уже используется это значение.
То есть в операции x -=10;, значение переменной х уже 20, поэтому alert выводит на экран 10, т.к. 20-10=10.

Присвоение по цепочке

Давайте рассмотрим еще один интересный способ - цепочку присвоений.
Созданим переменный a,b,c и присвоим им одно значение:
let a, b, c;
a = b = c = 2 + 2;

alert( a ); // 4
alert( b ); // 4
alert( c ); // 4
Такое присваивание работает справа налево.
Сначала вычисляется самое правое выражение 2 + 2, и затем результат присваивается переменным слева: c, b и a.
В конце у всех переменных будет одно значение.
Опять-таки, чтобы код читался легче, лучше разделять подобные конструкции на несколько строчек:
c = 2 + 2;
b = c;
a = c;

Операторы инкремента и дискримента

Операции инкремента и декремента являются унарными и производят увеличение и уменьшение значения операнда на единицу.
В качестве операнда может быть переменная, элемент массива, свойство объекта.
Чаще всего такие операции используются для увеличения счетчика в цикле.

Тут сделаем небольшое отступления для ВАЖНОГО пояснения.

Существует три вида/формы записи математических и логических формул:

  • Инфиксная - это форма записи математических и логических формул, в которой операторы записаны в инфиксном стиле между операндами, на которые они воздействуют (например, 2 + 2, или х+y).
    Задача разбора выражений, записанных в такой форме, для компьютера сложнее по сравнению с префиксной (то есть + 2 2) или постфиксной (2 2 +).
    Однако эта запись используется в большинстве языков программирования как более естественная для человека.

    В инфиксной нотации, в отличие от префиксной и постфиксной, скобки, окружающие группы операндов и операторов, определяют порядок, в котором будут выполнены операции.
    При отсутствии скобок операции выполняются согласно правилам приоритета операторов.

    Инфиксная запись может отличаться от функциональной, где имя функции описывает какую-то операцию, а её аргументы являются операндами.
    Примером функциональной записи может быть S(1,3) в которой функция S означает операцию сложения: S(1,3) = 1+3 = 4.

  • Префиксная - это когда оператор идёт перед переменной(т.е. ++х, где х - переменная, а ++ - операнд).

  • Постфиксная - это когда оператор идет после переменной(т.е. х++, где х - переменная, а ++ - операнд).

Отличие ПРЕФИКСНОЙ формы от ПОСТФИКСНОЙ в том, что:
- префиксная форма возвращает новое значение(т.е. уже измененное число)
- постфиксная форма возвращает старое значение(т.е. число до его увеличения или уменьшения).

Поэтому ВАЖНО запомнить, что при префиксной форме, мы сразу увидим новое значение
(пример: переменная х = 10, alert(++x), как результат увидим 11, т.е. значение увеличилось сразу на 1),
а вот в постфиксной форме, увидим старое значение еще до увеличения или уменьшения
(пример: переменная х = 10, alert(x++), как результат увидим 10, но если будем проводить после этого с переменной х будем проводить какие то другие операции то её значение уже будет 11).
При этом у постфиксной формы есть еще один важный момент, а именно при выполнении нескольких операций с нашей переменной x, то единица будет добавлена в конце по окнчанию операции
(пример:
x = 5;
s=x++*2;
alert(x+'\n'+s);
на экране увидим значение переменной х = 6 s= 10, т.е. сначала выполняется операция умножения, а после в переменной 'y' сохраняется увеличенное на 1 значение).

Теперь переходим непосредственно к понятию операторов инкремента и декремента.
инкримент (++) - это операция имеет синтаксис a++ или ++a и увеличивает переменную а нa единицу.
декремент(--) - это операция имеет синтаксис a-- или --a и уменьшает переменную а нa единицу.

Пример:
let x=1;
++х;
alert(x);
let v=4;
v++;
alert(v);
то увидим на экране значение переменной x = 2 и значение переменной v=5.
Однако если мы сделаем код записи немного другой, а именно:
let x=1;
alert(++x);
let v=4;
alert(v++);
то увидим несколько иной результат, так значение переменной х = 2, а вот значение переменной v = 4, поскольку это постфиксная форма записи и функция возвращаем нам значение переменной до его увеличения на 1.

Аналогично если мы повторим тот же самый код только с дексрементом, то значение переменных уменьшаться на 1.
Привер:
let x=1;
--х;
alert(x);
let v=4;
v++;
alert(v);
на экране в модальном окошке видим значение переменной х = 0, а значение переменной v = 3.

Операторы сравнения

Операторы сравнения предназначены для сравнения двух операндов. Если сравнение верное, то в качестве результата этой операции возвращается true. В противном случае, значение false.

В JavaScript выделяют следующие операторы сравнения:

== (равенств/ровно) 10==20 - false;
!= (не равенство/не равно) "20"===20 - false;
=== (строгое равенство/строго равно) 10!=20 -true;
!== (строгое не равенство/строго не равно) 20!=20 - false;
> (больше чем) 20>10 - true;
>= (больше или равно) 20>=10 - true;
< (меньше чем) 20'<'10 - false;
<= (меньше или равно) 20<=10 - false.

Равенство (==) и неравенство (!=)

Оба оператора принимают операнды любого типа и, если операнды равны, оператор равенства == возвращает true, а оператор неравенства != возвращает – false.

Получив операнды разных типов, эти операторы пытаются привести их к одному типу и сравнивают уже преобразованные значения.

При преобразовании типов операторы равенства == и неравенства != руководствуется следующими правилами:

  • Если какой-либо операнд число или булево значение, то операнды преобразуются в числа, при этом true преобразуется в 1, а false преобразуется в 0.
  • Если какой-либо операнд строка – второй преобразуются в строку.
  • Если одно из значений представляет собой объект, а другое – число или строку, то объект преобразуется в элементарный тип.

При сравнении операндов также применяются свои правила:

  • Если одно из значений равно null, а другое – undefined, то они равны.
  • Значения null и undefined не преобразуются для сравнения ни в какие другие значения.
  • Значение NaN считается не равным никакому другому значению, включая само себя.
    При его наличии оператор равенства всегда возвращает false, а оператор неравенства – true.
  • Если оба операнда являются объектами, то они сравниваются, чтобы выяснить, один ли это объект.
    Если да, возвращается true, иначе – false

Примеры:

alert((null == undefined) + ' это результат (null == undefined)'); // true
alert((NaN == NaN) + ' это результат (NaN == NaN)'); // false
alert((NaN != NaN) + ' это результат (NaN != NaN)'); // true
alert((false == 0) + ' это результат (false == 0)'); // true
alert((true == 1) + ' это результат (true == 1)'); // true
alert(("1" == true) + ' это результат ("1" == true)'); // true
alert(("" == false) + ' это результат ("" == false)'); // true
alert((undefined == 0) + ' это результат (undefined == 0)'); // false
alert((null == 0) + ' это результат (null == 0)'); // false

Строгое равенство (===) и строгое неравенство (!==)

Операторы строгих равенства и неравенства, известные также как операторы идентичности (===) и неидентичности (!==), делают то же самое, что и обычные операторы равенства и неравенства, но не преобразуют типы операндов перед их сравнением.

Оператор идентичности (===) возвращает true, только если операнды равны без преобразования:

  • Если оба операнда имеют примитивные типы и содержат одинаковые значения, они считаются равными.
  • Два операнда считаются равными, если оба ссылаются на один и тот же объект.
    Если же они ссылаются на разные объекты, они не равны, даже если эти объекты имеют идентичные свойства.
  • Оператор идентичности считает значение NaN не равным никакому другому значению, включая само себя.
    При его наличии оператор идентичности всегда возвращает false.

В противоположность оператору идентичности (===) оператор неидентичности (!==) возвращает true, если без преобразования операнды не равны.

Пример:

alert((null === undefined) + ' это результат (null === undefined)'); // false
alert((NaN === NaN) + ' это результат (NaN === NaN)'); // false
alert((NaN !== NaN) + ' это результат (NaN !== NaN)'); // true
alert((false === 0) + ' это результат (false === 0)'); // false
alert((true === 1) + ' это результат (true === 1)'); // false
alert(("1" === true) + ' это результат ("1" === true)'); // false
alert(("" !== false) + ' это результат ("" === false)'); // true
alert((undefined !== 0) + ' это результат (undefined !== 0)'); // true
alert((null === 0) + ' это результат (null === 0)'); // false

Выражение null == undefined истинно, потому что значения равны, но null === undefined ложно, потому что типы этих значений разные.

Сравнение строк

Если оператор отношения применяется к двум строкам, то сравнение происходит побуквенно/посимвольно.
То есть сравнение происходит так: каждый символ (буква) первой строки сравнивается с соответствующим символом второй строки и так, до того момента, пока символы не будут разными, и тогда какой символ больше – та строка и больше.
Если же в одной из строк закончились символы, то считаем, что она меньше, а если закончились в обеих – строки равны.

Теперь разберем, как происходит сравнение каждого символа.
Каждый символ переводится в числовое значение, согласно кодировке Unicode, и суть сравнения символов сводиться к сравнению двух чисел.
Например, десятичные (Dec) значения символов кирилицы лежат в диапазоне от 1024 до 1280.

Пример:

let str1 = "Апельсин";
let str2 = "абрикос";

Начинаем сравнивать первые символы: str1[0] (буква "А") и str2[0] (буква "а").

В JavaScript, как и в других языках програмирования, есть метод для получения цифрового кода из символа:

str.charCodeAt(pos);

Метод возвращает код символа расположенного на позиции pos в строке str.
Отсчет позиций символов в строке начинается с нуля.

Таким образом, наш код будет следующим:

let str1 = "Апельсин";
let str2 = "абрикос";
alert( str1.charCodeAt(0) ); // 1040, код 'А'
alert( str2.charCodeAt(0) ); // 1072, код 'а'
alert( str1.charCodeAt(0) < str2.charCodeAt(0) ); // true (1040 < 1072)
document.write( (str1 < str2) + '<'br>' ); // true
или последнюю строку кода можно записать иначе
alert( (str1 < str2) + ' это код сравнения переменных (str1"<"str2)' ); // true

Если бы первые символы строк были равны, то дальше сравнивались бы следующие символы попарно.

В этом примере строка "Апельсин" считается меньше, чем строка "абрикос", потому что буква "А" имеет код 1040, а буква "а" – 1072.
Причина в том, что коды прописных букв всегда меньше, чем строчных.

Чтобы правильно сравнить строки по алфавиту, необходимо перед сравнением преобразовать оба операнда в один регистр (например, в нижний):

let str1 = "Апельсин";
let str2 = "абрикос";
str1 = str1.toLowerCase(); // "апельсин"
str2 = str2.toLowerCase(); // "абрикос"
alert( (str1 < str2) + ' это сравнение str1"<"str2' ); // false

После преобразования в нижний регистр слово "апельсин" становится больше, чем слово "абрикос", потому что вторая буква "п" имеет код 1087, а вторая буква "б" переменной str2 – 1073.

Обычно, буква расположенная в алфавитном порядке раньше имеет меньший числовой код, чем буква (в том же регистре) расположенная в алфавитном порядке после неё.

Не менее интересная ситуация складывается при сравнении чисел, предствленных в строковой форме, например:

script>
alert( "15" < "5" ); // true
/script>

Здесь "15" < "5", потому что числа, представленные в виде строк, сравниваются как строки. Строковые операнды, как вы уже знаете, сравниваются по кодам знаков (код знака "1" – 49, код знака "5" – 53 ).

Если хотя бы один операнд изменить на число, то другой будет преобразован к числу:

script>
alert( "15" < 5 ); // false
/script>

В этом примере строка "15" преобразуется в число 15, а затем производится сравнение полученных чисел.

Когда строка сравнивается с числом, она преобразуется в число и выполняется числовое сравнение операндов.

В случае, когда строку невозможно преобразовать в число, она преобразуется в NaN.
Как правило, в результате любой операции сравнения с операндом NaN получается false:

script>
alert( "b" < 5 ); // false
alert( NaN < 5 ); // false
alert( NaN >= 5 ); // false
/script>

Подведем итоги
  • Операторы сравнения сравнивают значения операндов и возвращают логическое значение – true или false в зависимости от результатов проверки.
  • При сравнении операнды, имеющие значения разных типов, приводятся к числовому значению, за исключением сравнения на идентичность === и неидентичность !==.
  • При сравнении строк, сравнение происходит побуквенно: каждый символ переводится в числовое значение, согласно кодировке Unicode, и суть сравнения символов сводиться к сравнению двух чисел.
  • Код буквы в нижнем регистре всегда больше, чем код буквы в верхнем регистре.
  • Код цифры находящейся в строке всегда меньше кода буквенного символа в любом регистре.
  • Как правило, код буквы, расположенной в алфавитном порядке раньше имеет меньшее значение, чем код буквы (в том же регистре), расположенной в алфавитном порядке после неё. Операторы сравнения считают значение NaN не равным никакому другому значению, включая само себя.
  • Значения null и undefined НЕ СТРОГО равны == друг другу.
  • При преобразовании в число null становится 0, а undefined становится NaN.
